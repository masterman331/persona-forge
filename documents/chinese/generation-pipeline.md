# 生成管线

当你运行 `python main.py` 时，系统会经过**5个生成阶段**来创建一个完整的人格。每个阶段都建立在前一个阶段之上，逐步增加深度和细节。

---

## 阶段1：访谈

**文件：** `generators/interview.py`
**目的：** 收集关于你想要创建的人格的信息。

访谈引擎会问一系列问题来收集种子数据。你提供的细节越多，生成的人格就越丰富。

### 基本问题
- 姓名（名+姓）
- 当前年龄
- 出生地点（城市、州/国）
- 性别
- 任何额外备注（自由格式）

### 访谈选项
你可以选择：
1. **完整访谈** — 回答关于人格生活、关系、个性、经历的详细问题
2. **跳过访谈** — 只使用基本信息，让系统填充其余部分

### 风格选择
选择人格人生故事的叙述风格：
- 自动检测（系统根据人格决定）
- 多种预定义风格
- 自定义风格（描述你自己的）

额外备注字段是魔法发生的地方——"让TA因为童年事件而恐惧狗"或"他们秘密想成为音乐家但父母不允许"给了系统贯穿整个人格的特定线索。

---

## 阶段2：蓝图

**文件：** `generators/blueprint.py`
**目的：** 从访谈数据生成结构化的人生时间线。

蓝图定义了人格生活的**时代**——塑造了TA们是谁的主要章节。

### 蓝图结构

```json
{
  "life_eras": [
    {
      "era_number": 1,
      "label": "The Tooting Blur",
      "age_range": "0-6",
      "emotional_tone": "chaotic warmth",
      "chapters": [
        {
          "outline": {
            "time_span": "ages 0-2",
            "theme": "The loud beginning"
          }
        }
      ]
    }
  ]
}
```

每个时代有**6个章节**（可通过 `config.py` 中的 `CHAPTERS_PER_ERA` 配置）。

### 用户审查
蓝图生成后，你可以在继续之前**审查和批准**它。你可以看到时代结构并进行调整。

---

## 阶段3：深度生成

**文件：** `generators/deep_generator.py`
**目的：** 将每个时代的每个章节扩展为丰富、详细的内容。

这是人格的人生故事**完全实现**的地方。每个章节都会获得：

### 章节深度潜入

```
┌──────────────────────────────────┐
│             章节深度潜入                            │
│                                                          │
│  日常生活作息                                     │
│  ├── TA们典型的一天是什么样的                  │
│  ├── 放学前/后做了什么                          │
│  └── 周末模式                                    │
│                                                          │
│  关键场景（每章节5个）                        │
│  ├── 地点：发生在哪里                          │
│  ├── 发生了什么：事件本身                        │
│  ├── 感受：情感体验                            │
│  └── 感官片段：特定的感觉记忆                    │
│                                                          │
│  关系演变                                       │
│  ├── 人物：谁                                  │
│  └── 变化：关系如何改变                        │
│                                                          │
│  知识获取                                       │
│  ├── 事物：学到了什么                          │
│  └── 方式：知识来源                            │
│                                                          │
│  观点快照                                       │
│  ├── 观点：相信什么                          │
│  └── 原因：为什么相信                          │
│                                                          │
│  佚语和语言                                   │
│  ├── 短语/模式：如何说话                      │
│  └── 来源：从哪里学到的                      │
│                                                          │
│  尴尬时刻                                       │
│  └── TA们想起来仍会盯脚的时刻                    │
└──────────────────────────────────┘
```

每个章节有**5个关键场景**（可通过 `SCENES_PER_CHAPTER` 配置），每个场景都有地点、事件、情感内容和感官细节。

这个阶段最消耗 API——它为每个时代的每个章节生成详细内容。

---

## 阶段4：记忆蒸馏

**文件：** `generators/distillery.py`
**目的：** 从深度生成输出中提取和分类记忆。

蒸馏器读取丰富的人生故事并将其**蒸馏**成结构化的记忆文件。它遍历所有场景、事件和情感时刻，并对它们进行分类：

### 记忆类别

| 类型 | 描述 | 最大数量 | 例子 |
|------|-------------|-----------|----------|
| 核心 | 塑造了TA们是谁的人生定义性时刻 | 15 | 父母离婚的那天；赢得才艺表演的时候 |
| 标志性 | TA们告诉别人的故事 | 30 | 夏令营的那次；那次出错的自驾游 |
| 感官 | 特定的基于感觉的闪回 | 无限 | 祖母厨房的气味；学校屋顶雨声的声音 |
| 环境 | 被提醒时记起 | 无限 | 高中第一天；总是挥手的邻居 |
| 休眠 | 几乎无法访问，被提醒时才会想起 | 无限 | 已经忘记的老师名字；童年卧室的颜色 |
| 虚假 | TA们相信但其实不真实的事情 | 无限 | "我爸是冠军拳击手"（其实不是）；"我从没去过苏格兰"（其实去过） |

### 如何运作

1. 蒸馏器读取所有时代文件
2. 识别情感重要的时刻
3. 根据重要性级别对每个时刻进行分类
4. 添加元数据：情感权重、触发词、信心级别
5. 写入结构化的记忆文件

### 虚假记忆

最有趣的功能之一：蒸馏器**故意创建虚假确定性**。这些是人格有信心地相信但其实错误的事情。这模仿了真实的人类记忆，后者是出了名的不可靠。

人格永远不知道这些是虚假的。他们以同样的信心换卫这些，就像换卫真实记忆一样。这对于真实感至关重要——真人经常搞错事情并且自己不知道。

## 阶段5：人格合成

**文件：** `generators/synthesis.py`
**目的：** 将所有内容组合成最终的人格文件结构。

合成阶段取用深度生成输出和蒸馏输出，产生最终的结构化文件：

### 生成的内容

1. **身份** (`identity.json`) — 姓名、年龄、性别、出生地点、写作风格
2. **声音** (`language/voice.json`) — 句子结构、词汇水平、幽默风格、脆弱表达、紧张口吃、发消息 vs 说话风格
3. **短语** (`language/phrases.json`) — 常用短语、口头禅
4. **佚语** (`language/slang.json`) — 时代特定佚语及其来源
5. **引用** (`language/references.json`) — TA们会使用的流行文化引用
6. **模式** (`psychology/patterns.json`) — 行为模式（如何处理冲突、无聊时做什么等）
7. **沟通** (`psychology/communication.json`) — 沟通风格详情
8. **内心世界** (`psychology/inner_life.json`) — 循环思维、核心恐惧、秘密欲望、隐藏的骄傲、理想与现实的差距、秘密（按亲密度分级）
9. **矛盾** (`psychology/contradictions.json`) — TA们自相矛盾的方式及例子
10. **知识** — 专业知识、随性知识、盲区、错误信念
11. **观点** (`opinions/timeline.json`) — 带有时代、原因和确定性的观点
12. **关系** — 家庭、朋友、浪漫史及完整详情
13. **一致性账本** (`_ledger.json`) — 用于一致性检查的交叉引用

### 一致性账本

一致性账本 (`core/consistency_ledger.py`) 跟踪人格不同部分之间的交叉引用。如果一段记忆引用了一个人，账本确保这个人存在于关系文件中。如果一个观点与另一个观点矛盾，账本会标记它（或将其视为现实主义的矛盾而接受）。

## 提示词构建器

**文件：** `generators/prompt_builder.py`

提示词构建器为每个生成阶段构建 LLM 提示词。它将 `config.py` 中的**180条行为规则**融入其中，以确保生成的内容遵循项目对真实人类行为的哲学。

关键方面：
- 关于具体性的规则（不是"一家餐厅"而是"河滨公园附近那家 Applebee's"）
- 关于不完美记忆的规则（30% 的日期在生成后变得模糊）
- 关于流行文化嵌入的规则（每个时代的文化氛围是什么）
- 关于无聊的规则（不是每个月都充满戏剧性）
- 关于尴尬的规则（每个人格至少5-10个让TA乐的眨眼的尴尬记忆）

## Token 使用量

生成管线**非常**消耗 API。这里是大概的分解：

| 阶段 | API 调用 | 估计 Tokens |
|-------|-----------|------------------|
| 访谈 | 0（仅用户输入） | 0 |
| 蓝图 | 1-3 | 5,000-15,000 |
| 深度生成 | 每章节1次 | 50,000-200,000 |
| 蒸馏 | 每时代1-2次 | 20,000-60,000 |
| 合成 | 5-10 | 30,000-80,000 |
| **总计** | **20-50+** | **100,000-350,000+** |

这就是为什么默认模型 (`zai:cloud`) 是本地模型——使用付费 API 生成人格会很贵。一次完整的人格生成很容易消耗 200,000+ tokens。
