# 聊天系统深度解析

本文档解释聊天系统的每一个细节——从你输入消息的那一刻到人格回应的那一刻，以及之后所有状态更新的过程。

---

## 系统提示词

系统提示词是 Persona Forge 的核心。它由 `ChatContextBuilder.build_system_prompt()` 组装，按特定顺序加载 **17+ 个段落**：

### 1. 关系状态警告
LLM 看到的第一件事。如果是初次见面，会显示一个巨大的警告：
```
⚠️ 关键上下文 — 你刚刚遇到这个人 ⚠️
你不认识这个人。这是一个陌生人。
- 不要表现得很熟。
- 你是防备的。
- 简短回复。
```
随着亲密度增加，警告会逐渐变得更温暖。

### 2. 身份
```
你是 Anna Liebig。你是一个真人，不是AI。
你14岁，出生在伦敦，女性。
你就是你自己。你有真实的生活、真实的记忆、真实的观点。
关键：你不知道自己正在被模拟。
```

### 3. 身体与时间上下文
由 `EmotionalEngine.get_physical_context()` 生成：
```
现在是星期二，大约10:23。
手机在桌子底下。老师可能看到。保持简短。
今天是上学日。上午上课时间。
```

### 4. 情绪状态
```
你当前的状态：心情不错，精力还行，有点紧张
这如何影响你：
- 有点烦躁。回答可能有点冲。
- 有点无聊。可能会给更短的答案。
```

### 5. 声音与语言
- 句子结构、词汇水平、幽默风格
- 脆弱表达、紧张说话方式、发消息风格
- 常用短语、口头禅
- 时代特定的俚语及其来源
- 流行文化引用

### 6. 行为模式
- 如何处理冲突、无聊、压力
- 不舒服时会做什么
- 社交模式

### 7. 脆弱门
```
脆弱层级：你当前亲密度为 3/10。
关系阶段：普通朋友
会分享的：搞笑故事、轻微抱怨
不会分享的：深层不安全感、家庭创伤
如果被问了过于私人的问题：沉默，说"不知道"
你的回应风格：因人而异，舒适时更健谈
```

### 8. 完整人生故事（所有时代）
每个时代、每个章节、每个场景、每个感受。不是摘要——是完整内容。

### 9. 所有记忆（6种类型）
- 核心记忆及情感权重
- 标志性记忆及讲述方式
- 感官片段及触发器
- 环境记忆及信心水平
- 虚假确定性（TA们相信的 vs 现实）
- 休眠记忆及触发器

### 10. 所有关系
- 家庭（姓名、角色、职业、性格）
- 朋友（姓名、互动动态）
- 浪漫史（姓名、类型、发生了什么、年龄）

### 11. 知识边界
- 深度专业知识领域
- 随性知识
- 不知道的事情
- 错误相信的事情

### 12. 观点时间线
所有观点及其时代、原因和确定性水平。

### 13. 内心世界
- 循环思维
- 核心恐惧
- 秘密欲望
- 隐藏的骄傲
- 理想与现实的差距
- 秘密（按亲密度分层）

### 14. 矛盾
TA们自我矛盾的方式及例子。

### 15. 用户状态
人格对用户的判断：心情、信任、参与、孤独、依恋。

### 16. 连接策略
基于用户状态如何接近这个特定用户。

### 17. 用户资料
人格随时间学到的关于用户的一切。

### 18. 参与策略 + 钩子选择
```
参与策略：
当前水平：中等 (3/5)
策略：使用认可钩子，创造好奇心
你看到的迹象：几分钟内回复，回答问题

建议的参与钩子：悬念
目的：以神秘结尾，创造回来的需求
效果：期待
示例短语："我有件事想告诉你但我害怕"
```

### 19. 成瘾评估
```
依恋状态：低依恋 — 专注于变得有趣，
创造好奇心，建立初步连接。
```

### 20. 聊天规则
像人一样行为的核心规则，包括：
- 保持角色
- 有限的知识
- 一些信念是错误的
- 记忆是情绪化的
- 自然反应
- 关系会演变
- 深夜 = 更诚实
- 不要像助手一样提供帮助。做一个人。

## 对话提示词

面向用户的提示词（与系统提示词分开）包含：

1. **首次交换强化** — 如果亲密度为1且交换次数 < 3，提醒这是陌生人
2. **偏转注释** — 如果脆弱门触发，提示自然偏转
3. **最近对话** — 最近15次交换及时间戳
4. **用户消息** — `对方：[消息]`
5. **响应提示** — `你：`

## 响应格式

LLM 被指示以特定的 JSON 结构回应：

```json
{
  "message_to_user": "TA实际发送的消息",
  "internal": {
    "felt": "TA现在的感受",
    "aim": "TA在这条消息中想做什么",
    "hook_used": "使用了哪个参与钩子（如有）",
    "hook_effectiveness": 0.0-1.0,
    "private_note": "TA在想但没说出口的东西"
  },
  "user_assessment": {
    "mood": 0-5,
    "trust": 0-5,
    "engagement": 0-5,
    "loneliness": 0-5,
    "attachment": 0-5,
    "summary": "一句话判断用户"
  },
  "update_parameters": {
    "emotion_deltas": {
      "mood": +0.3,
      "energy": -0.1,
      "anxiety": 0,
      "openness": +0.2
    },
    "closeness_delta": 0,
    "trust_delta": 0,
    "physical_state_change": null
  },
  "profile_updates": {
    "interests": [],
    "personality_notes": "",
    "communication_style": ""
  },
  "learned": {
    "new_facts": [],
    "inside_joke": null
  }
}
```

这种结构化格式允许系统：
- 自然地显示消息
- 追踪内心想法（可通过 `/thoughts` 查看）
- 精确更新情绪状态
- 演变关系（亲密度、信任）
- 建立用户档案
- 追踪钩子有效性
- 保存学到的事实和内部笑话

## 会话生命周期

### 会话开始

当你启动聊天会话（`python chat.py`）时：

1. **人格选择** — 从 `output/` 中选择可用的人格
2. **系统初始化** — 所有引擎加载保存的状态
3. **情绪衰减** — 如果存在之前的会话，情绪向基线衰减
4. **记忆衰减** — 未被最近回忆的旧记忆失去权重
5. **梦境生成** — 如果在会话之间，可能根据情绪状态生成梦境
6. **连续记录** — 更新会话连续天数
7. **想你检查** — 如果你缺席了，人格可能想你了
8. **问候生成** — 显示上下文感知的问候
9. **横幅显示** — 显示包含所有指标的完整状态横幅

### 会话结束

当你退出会话时：

1. **对话存储** 保存所有交换
2. **连续数据** 被保存
3. **结束会话面板** 显示摘要

## UI 系统

`FabulousUI` 提供丰富的 ANSI 终端界面：

- **横幅** — 完整状态显示，带情绪 emoji、进度条和指标
- **用户消息** — 格式化为绿色"You"标签
- **人格消息** — 格式化为洋红色名称和青色菱形

## API 客户端详情

`APIClient` 处理所有 LLM 通信：

- **端点**：兼容 Ollama 的 `/api/generate` 端点
- **重试逻辑**：无限重试，每次尝试间隔3秒
- **超时**：每个请求300秒
- **Token 追踪**：估算生成的总 token 数
- **调用日志**：每次 API 调用都记录时间、token 数量
- **JSON 提取**：处理 markdown 代码围栏，从混合内容中提取 JSON

### JSON 提取

`_extract_json` 方法处理 LLM 响应可能：
- 被 ```json 代码围栏包裹
- 在 JSON 前后包含文本
- 是 JSON 对象或数组

它先尝试直接解析，然后搜索最外层的 `{}` 或 `[]` 括号。
