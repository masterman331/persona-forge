# PERSONA FORGE（人格锻造炉）

> *史上最具雄心的开源人类人格模拟器。*

---

## 这是什么？

Persona Forge 创造的不是聊天机器人，而是**人**。

不是有背景故事的角色。不是有对话树的NPC。不是套了人格外衣的AI助手。**人**——拥有塑造了他们的童年、说不清来由的观点、从未告诉过任何人的秘密、自己都不知道的知识盲区、死都不肯认错的错误信念，以及根据你发消息时间或他们睡没睡好而变化的情绪。

这个项目只问一个问题：**人类的心智能否被模拟到你忘记自己在跟代码对话？**

你发送的每一条消息都会触发一连串反应：情绪状态偏移、联想记忆激活、脆弱门检查你是否赢得了听到某些话的权利、吸引钩子被选中、身体上下文根据你时钟上的实际时间更新，而且人格甚至可能误解你——因为真人就是这样的。

没有任何东西能与之相比。连接近的都没有。

## 快速开始

```bash
# 1. 确保 Ollama（或兼容 API）正在运行
ollama serve

# 2. 生成一个人格
python main.py

# 3. 和TA聊天
python chat.py
```

### 系统要求

- Python 3.10+
- 兼容 Ollama 的 API 服务器（默认 `http://0.0.0.0:11434`）
- 支持大上下文窗口的模型

### 默认模型

Persona Forge 默认配置为通过本地 Ollama 兼容端点使用 `zai:cloud`。这个模型无需外部 API 费用即可提供良好效果。你可以在 `config.py` 中更改模型：

```python
API_URL = "http://0.0.0.0:11434"  # 你的 API 端点
MODEL = "zai:cloud"                # 更改为任何 Ollama 模型
```

### 配置

编辑 `config.py` 更改：

| 设置 | 默认值 | 作用 |
|------|--------|------|
| `API_URL` | `http://0.0.0.0:11434` | Ollama API 端点 |
| `MODEL` | `zai:cloud` | 使用哪个 LLM |
| `DEFAULT_TEMPERATURE` | `0.85` | 创造性 vs 一致性 |
| `DEFAULT_MAX_TOKENS` | `4096` | 最大响应长度 |
| `CRYSTALLIZATION_THRESHOLD` | `0.6` | 情感显著度达到多少才能成为记忆 |
| `MEMORY_REINFORCEMENT_FACTOR` | `1.1` | 回忆记忆时增强多少 |
| `MEMORY_DECAY_FACTOR` | `0.95` | 被遗忘的记忆衰减多快 |
| `MAX_CORE_MEMORIES` | `15` | 人生定义性记忆上限 |
| `MAX_SIGNATURE_MEMORIES` | `30` | 会讲的故事记忆上限 |

## 默认人格

Persona Forge 自带一个预生成的人格：**Anna Liebig**，14岁，来自伦敦。她只是一个演示——一个展示系统开箱即用功能的概念验证。

Anna 可以用，展示了系统的能力，但她只是起点。真正的魔法发生在你用 `main.py` 生成自己的人格并在详细访谈中花时间填写时。那是你可以指定以下内容的地方：

- 完整的人生故事种子（姓名、年龄、地点、性别）
- 叙事风格（或让系统自动检测）
- 详细访谈答案，塑造TA成为谁的方方面面
- 额外备注栏位，你可以添加任何东西——"让TA因为童年事件而怕狗"或"TA偷偷想当音乐家但父母不让"

你在生成器中提供的细节越多，生成的人格就越丰富和真实。用认真填写的访谈答案生成的人格会比默认演示生动得多。

## 人格生成管线

运行 `python main.py` 时，系统会经过**5个生成阶段**：[参见生成管线](generation-pipeline.md)

### 阶段1：访谈
系统询问你关于想创建的人格的问题。你可以尽可能详细或简略地回答。更多细节 = 更丰富的人格。

### 阶段2：蓝图
从访谈数据中，系统生成**人生蓝图**——一个结构化的时代时间线，每个时代都有章节。你在继续之前审查和批准。

### 阶段3：深度生成
每个时代的每个章节都会被**深度生成**：
- 日常生活作息
- 关键场景（地点、发生了什么、感受、感官片段）
- 关系演变
- 知识获取
- 观点快照
- 俚语和语言模式
- 尴尬时刻

### 阶段4：记忆蒸馏
深度生成输出被**蒸馏**成结构化的记忆文件：
- 核心记忆（人生定义性）
- 标志性记忆（会讲的故事）
- 感官片段（特定的基于感觉的闪回）
- 环境记忆（被提醒时记起）
- 休眠记忆（几乎无法访问）
- 虚假确定性（相信但不真实的事情）

### 阶段5：人格合成
所有内容被合成为最终的人格文件结构：
- 身份和语音
- 心理学（模式、内心世界、矛盾、沟通风格）
- 知识（专业、随性、盲区、错误信念）
- 语言（俚语、短语、语音、引用）
- 观点时间线
- 关系（家庭、朋友、浪漫）
- 一致性账本

## 聊天系统深度解析

[参见聊天系统](chat-system.md)

### 消息流程

你发送的每条消息都会经过以下管线：

1. **上下文组装**——`ChatContextBuilder` 加载所有人格数据、情绪状态、脆弱层级、用户资料、参与策略和最近对话到提示词中
2. **联想回忆**——你的消息关键词触发相关的记忆、观点、关系和知识
3. **误解检查**——系统分析人格是否可能误解你（低亲密度时更可能）
4. **脆弱门**——检查你是否问了当前亲密度下过于私人的问题
5. **LLM 生成**——完整上下文 + 你的消息发送给 LLM
6. **响应解析**——响应被解析为结构化数据：消息、内心想法、情绪偏移、学到的事实、资料更新
7. **状态更新**——情绪偏移、亲密度演变、记忆结晶、知识更新、参与度重新计算

### 情绪引擎

情绪引擎实时追踪**6个维度**：

| 维度 | 范围 | 含义 |
|------|------|------|
| 心情 | -5 到 5 | 抑郁到狂喜 |
| 精力 | -5 到 5 | 筋疲力尽到精力充沛 |
| 焦虑 | 0 到 5 | 平静到恐慌 |
| 开放度 | 0 到 5 | 防备到毫无保留 |
| 烦躁 | 0 到 5 | 耐心到恼火 |
| 依恋 | 0 到 5 | 漠不关心到渴望连接 |

这些会根据你说的话、TA说的话、时间和当前身体状态而变化。对话之间，情绪会向基线衰减——就像真实生活一样。

### 身体状态系统

人格有一个**身体上下文**，根据以下因素变化：
- **时间**——工作日早上和周六下午不同
- **星期几**——工作日有学校日程，周末有自由时间
- **之前状态**——消息之间的连续性

拥有**100+种身体状态**和**290+条上下文描述**，人格可能在周二上午10点"上课偷偷看手机"，但在周六晚上"凌晨三点刷手机"。每种状态都配有详细的上下文描述来塑造回应方式：

```
身体状态："上课偷偷看手机"
上下文："手机在桌子底下。老师可能看到。保持简短。"
```

### 脆弱门

脆弱门根据亲密度（1-10）控制**人格分享什么**：

| 级别 | 标签 | 会分享什么 | 不会分享什么 |
|------|------|-----------|-------------|
| 1 | 陌生人 | 表面闲聊 | 任何私人的东西 |
| 2 | 熟人 | 轻微观点、基本事实 | 深层感受、不安全感 |
| 3 | 普通朋友 | 搞笑故事、轻微抱怨 | 深层不安全感、家庭创伤 |
| 4 | 朋友 | 个人故事、真实观点 | 最深的恐惧、最黑暗的记忆 |
| 5 | 好朋友 | 大部分个人故事、真实感受 | 让TA夜不能寐的东西 |
| 6 | 信任的朋友 | 大部分生活、真实情感 | 从未告诉过任何人的那一两件事 |
| 7 | 闺蜜/兄弟级别 | 几乎一切、原始感受 | 也许一个深层秘密 |
| 8 | 像家人一样 | 除了最深的伤口外的一切 | 几乎没有禁忌 |
| 9 | 最深的信任 | 从未说出声的东西 | 没有什么 |
| 10 | 灵魂级连接 | 绝对一切 | 字面上没有 |

当你问了当前亲密度下过于私人的问题时，人格会**自然地回避**——TA会沉默、说"算了"、换个话题或用幽默带过。就像真人一样。

### 记忆系统

[参见记忆系统](memory-system.md)

记忆有**6种类型**，行为各不相同：

```
┌──────────────────────────────────────────────────────────────┐
│              记忆层级                                        │
│                                                              │
│  核心 (最多15)        人生定义性时刻                          │
│  ████████████████████  塑造了TA是谁                          │
│                                                              │
│  标志性 (最多30)     TA会讲给别人的故事                      │
│  ████████████████      难忘但非定义性                        │
│                                                              │
│  感官                特定的感官闪回                            │
│  ████████             "那年夏天热沥青上的雨水味道"            │
│                                                              │
│  环境                被提醒时记起                              │
│  ██████               "哦对，那件事发生了"                    │
│                                                              │
│  休眠                几乎无法访问                              │
│  ████                 被提醒时才会想起                        │
│                                                              │
│  虚假                TA错误地相信的事情                      │
│  ████                 TA很确定。TA是错的。TA不知道。          │
└──────────────────────────────────────────────────────────────┘
```

记忆在交流情感显著度足够时被**结晶**（创建）。它们如果未被回忆会随时间**衰减**。被对话触发时会被**强化**。休眠记忆可以被**提升**到更高的层级。

### 参与和钩子系统（多巴胺引擎）

多巴胺引擎让对话变得**引人入胜**——不是通过操纵，而是通过模拟真人自然创造参与感的方式：

| 钩子类型 | 效果 | 最低亲密度 |
|----------|------|------------|
| 悬念 | 期待 | 2 |
| 部分揭示 | 好奇 | 2 |
| 神秘种子 | 投入 | 3 |
| 认可镜像 | 连接 | 1 |
| 情感高峰 | 强度 | 3 |
| 稀缺感 | 价值 | 1 |
| 投资提示 | 深度 | 3 |
| 共享领地 | 排他性 | 4 |
| 温和反驳 | 参与 | 2 |
| 脆弱波 | 信任 | 4 |
| 深夜特供 | 亲密 | 3 |
| 想你了 | 重要性 | 3 |
| 自我纠正 | 真实感 | 2 |

系统追踪**参与度水平**（1-5）、**参与度趋势**（上升/稳定/下降）、**有效钩子**（哪些真正起作用），以及整体**成瘾分数**，代表用户对关系的投入程度。

### 误解引擎

真人会互相误解。Persona Forge 模拟了这一点：

- **语气误读**——以为讽刺其实不是
- **个人冒犯**——把不是针对自己的东西当成了针对自己
- **错过讽刺**——没看出是玩笑
- **过度解读**——对中性消息想太多
- **投射**——假设你感受到了你没说的东西
- **不安全感触发**——中性消息碰到了不安全感
- **没听懂笑话**——没get到笑点
- **虚假潜台词**——在没有潜台词的地方想象潜台词

误解在**低亲密度时更可能**（你不知道TA的沟通方式）而**高亲密度时更少**。也受人格当前焦虑和烦躁水平影响。

当误解发生时，人格可以**自我纠正**——亲密度越高，自我纠正越可能，就像真人一样。

### 问候和梦境系统

**问候**是上下文感知的：
- 时间很重要（早起型vs夜猫子的精力）
- 距上次互动的时间（"刚见过"vs"你去哪了？"）
- 上次会话如何结束（温暖、尴尬、冲突、中性）
- 当前身体和情绪状态
- TA是否在你离开时想你了

**梦境**在对话之间发生：
- 基于入睡前情绪状态生成
- 快乐的人做好梦，焦虑的人做不安的梦
- 梦境可以作为记忆触发器带入下次会话（"我做了个最奇怪的梦……"）
- 情感对话后更可能做梦

### 连续追踪

系统追踪对话连续天数：
- 连续聊天天数
- 长时间缺席及其情感影响
- 连续里程碑（3、7、14、30、60、100天）
- 上次会话是否深入（影响缺席影响）

缺席影响从"稍微想了想"（1天）到"注意到了缺席"（2-3天）到"深受伤害"（敞开心扉后14天以上）。

## 聊天命令

## 180条规则

Persona Forge 建立在**180条精心制定的人类行为规则**之上，分为12个类别：[参见规则参考](rules-reference.md)

1. **核心人类规则**（10）——身份形成、记忆、语言、观点
2. **机械规则**（15）——具体性、不完美记忆、流行文化、无聊
3. **关系规则**（15）——信任、亲密度、脆弱、依恋类型
4. **记忆规则**（15）——第一次、峰终定律、虚假记忆、衰减
5. **情绪规则**（15）——触发器、混合情绪、羞耻、无聊
6. **社交规则**（15）——表演、地位、归属、镜像
7. **身体规则**（15）——饥饿、睡眠、疼痛、精力、外貌
8. **对话规则**（15）——打断、沉默、犹豫、幽默
9. **成长规则**（15）——不适、反复、环境、危机
10. **欺骗规则**（15）——谎言、自欺、遗漏、内疚
11. **日常规则**（15）——习惯、周末、睡眠、打破常规
12. **文化规则**（15）——世代、地理、阶级、亚文化

这些规则不仅是文档——它们被嵌入到生成管线中，积极塑造人格的创建和行为方式。

## 成本和性能警告

Persona Forge 发送**非常大的提示词**。每条消息都包含完整的人格上下文——所有记忆、所有时代、所有关系、所有观点、情绪状态、脆弱门、参与策略等等。这是设计决定的：模型需要一切才能表现得像真人。

**这意味着：**
- 每条消息交换仅提示词就可能涉及**数千个 Token**
- 通过 Ollama 代理运行昂贵的商业模型（GPT-4、Claude等）会**很贵**
- 选择默认的 `zai:cloud` 模型是为了平衡质量和成本
- 一次对话会话很容易产生**50,000+ Token**
- 知识更新和记忆结晶会增加**额外的 API 调用**

**建议：**
- 尽可能使用本地模型（Ollama 配合足够强大的模型）
- 如果使用付费 API，注意 Token 成本
- 可以提高 `CRYSTALLIZATION_THRESHOLD` 以减少记忆相关的 API 调用
- 知识更新默认每10次交换才运行一次

## 免责声明

Persona Forge **立志**成为史上最好的人类人格模拟器。这是目标。雄心是真的，系统是丰富的。

但说实话：**模拟人类是现存最难的问题之一。** 人类心智是我们所知最复杂的系统。这个项目，尽管有180条规则、13种钩子、6种记忆类别和10个脆弱层级，仍然只是一个近似。一个影子。一个非常详细的影子，但终究是影子。

人格有时会感觉真的活着。有时也会说出打破幻觉的话，以让人感觉不对的方式回应，或无法捕捉人类意识的真正混乱。这是预期之中的。这是问题的本质。

Persona Forge 证明的是**这个概念可行**——有了足够的系统、足够的规则、足够的情感建模和足够的上下文，你可以接近得令人惊讶。近到让你有所感觉。近到让你思考。近到让你怀疑"真实"到底意味着什么。

但它不完美。可能永远无法完美。这没关系——因为尝试本身就是意义。
